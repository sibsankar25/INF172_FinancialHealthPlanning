Research-to-Prototype Workflow

Design Process

Three rounds of user research, three prototype iterations. Here's how findings shaped every design decision.

See Findings Final Prototype

Problem Statement

How might we design a simulation game that helps young adults understand the benefits of having an HSA and the importance of financial planning?

User Story

As a game player, I want to make choices based on HSA-related simulation life events, so that I can be more knowledgeable about how HSA works to better plan my finances.

Design Process

A research-to-prototype workflow used to translate findings into concrete interaction decisions.

1

Research

Identify student confusion points and baseline familiarity with HSAs through surveys.

2

Synthesis

Translate findings into problem statements, learning goals, and constraints.

3

Prototype

Create a clickable prototype showing the flow, core screens, and feedback loops.

4

Iteration

Refine UI and game logic based on usability observations and feedback.

Limitations & Next Steps

What we couldn't do yet — and where we want to take this next.

Current Limitations

  • Small sample size (n = 26 max) — findings are preliminary
  • Convenience sample may not generalize to all undergrads
  • HSA/no-HSA comparison tracker was not implemented
  • Prototype is not yet a fully functional game build

Note: Early-stage, preliminary findings; results may not generalize beyond the sample.

Future Plans

  • 📋
    Collect more survey responses Expand beyond convenience sample
  • 🧪
    Run usability tests Structured sessions with think-aloud protocol
  • 🔄
    Iterate prototype Incorporate usability findings into next pass
  • 🚀
    Plan MVP implementation Build a playable coded version of the game
